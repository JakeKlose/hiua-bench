# Paper-Writer Agent — Invocation Guide

A reusable subagent that produces a full CS321M-style research paper (long form + workshop cut) from a research memo, citing only sources it verifies via web search and fetch in the same session.

## Files

- `paper-writer.md` — the agent definition (system prompt, tool list, workflow)
- `README.md` — this file

## How to invoke

From a Claude session in this project, ask Claude to invoke the agent. Either of these will work:

> "Run the paper-writer agent on `hallucination_to_unauthorized_action_memo.md`. Output to a new folder `paper_drafts/v1/`."

> "Use the paper-writer subagent. Input memo: `hallucination_to_unauthorized_action_memo.md`. Working dir: `paper_drafts/v1/`. Target venues: CS321M term paper (long) and NeurIPS SafeGenAI workshop (short)."

Claude will then call the `Agent` tool with `subagent_type: "paper-writer"` and pass through your spec.

### Required invocation inputs

1. **Input memo path** — a Markdown file with the construct, lit-scan, and (optionally) design sketch. The current memo `hallucination_to_unauthorized_action_memo.md` is the canonical example.
2. **Working directory** — where the agent should write its outputs. Create a fresh one per run.
3. **Target venue(s)** — optional, defaults to CS321M long + generic safety workshop short.

### Outputs the agent produces

In the working directory:

- `paper_long.md` — 15–20 page manuscript
- `paper_workshop.md` — 8-page workshop cut
- `_citations.json` — verified bibliography (every cite traceable to a tool call)
- `_outline.md` — outline used
- `_review_notes.md` — agent's self-review, including `[CITE NEEDED]` count and weakest claims

## Expected runtime

A full run is long. Phase 1 (citation verification) alone typically takes 30–60 WebFetch calls. Plan on 20–40 minutes of agent runtime depending on how many sources the memo cites and how aggressively the agent expands the bibliography.

## What you MUST do after the run

1. **Citation verification pass.** Open `_citations.json` and spot-check 5–10 entries against the actual arXiv pages. The agent verifies as it writes, but a fresh-eyes audit catches anything that slipped.
2. **`[CITE NEEDED]` cleanup.** Search both manuscripts for this tag. The agent leaves these where it couldn't verify a claim; you decide whether to find a source or rewrite the claim.
3. **Construct-definition sanity check.** Read §3 of the long paper. If the construct definition has drifted from what you actually want to measure, the rest of the paper is downstream and needs realignment.
4. **Read `_review_notes.md`.** The agent flags its own weakest claims here. Treat this as a TODO list.

## Known limitations

- **The agent cannot read full PDF text.** WebFetch on arXiv abs pages gives the abstract and metadata. For specific numerical claims, the agent is verifying against abstracts, which sometimes omit the exact number. You may need to fact-check numbers against full papers manually.
- **No human-in-the-loop checkpoints.** You chose fully autonomous. If the agent's framing diverges from yours mid-paper, you find out at the end. Mitigate by reading the outline (`_outline.md`) first before reading the manuscripts.
- **Workshop cut is rewritten, not summarized.** This is better quality but means the two documents may make slightly different emphasis choices. Read both before deciding which to build on.
- **No figures or tables generated.** The agent produces Markdown text only. If the paper needs a comparison table of benchmarks or a validity-framework matrix, ask Claude in a follow-up turn to generate them from `_citations.json`.

## Iterating

After the first run:

- For targeted revisions: ask Claude to edit specific sections of `paper_long.md` directly. Don't re-run the agent for small changes — it will redo everything.
- For a major reframe: update the input memo, then re-run the agent to a new working directory (`paper_drafts/v2/`).
- For a different venue: the long form is structured so a new workshop cut can be generated by re-running just Phase 4 of the agent. Ask Claude to do this manually rather than re-running the full pipeline.

## One sentence of advice

The agent's output is a first draft, not a submission. Plan to spend at least a full day editing what it produces — the validity audit will need your own analytical voice, and the construct definition will need your refinements after you see it on the page.
